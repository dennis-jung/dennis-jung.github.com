Mockito on Android

Tweet

요즘 가장 큰 관심은, "어떻게 버그를 생성하지 않을까?" 입니다.

그래서 TDD부터 시작해 보기로 했습니다.

Assert만으로는 부족하다고 느껴졌고, mockito를 적용하기로 했습니다.

환경 셋팅

  1. mockito 다운로드
  2. 안드로이드에서 mockito를 사용하기 위해서는 dexmaker 라는 라이브러리가 추가로 필요하다.
    • dexmaker는 Dalvik VM을 대상으로 런타임 코드를 생성하는 자바 API이다.
    • dexmaker 다운로드
    • dexmaker-1.0.jar, dexmaker-mockito-1.0.jar 2개 모두 필요합니다.
  3. tests/libs 폴더에 다운로드 받은 jar 파일들을 넣으면 됩니다.

테스트 코드 작성

  • test 코드 상단에 Mockito를 import 하고, 테스트 코드를 작성합니다.
    import static org.mockito.Mockito.*;

    @SmallTest
    public void testMock() {
        List<?> mock = mock(List.class);
        when(mock.size()).thenReturn(1);

        assertEquals(mock.size(), 1);
        verify(mock).size();
    }

그 다음 mock을 이용해서 테스트 코드를 작성하면 됩니다.

Device에서 테스트 할때에는 문제가 없었는데, CI Emulator에서 테스트 코드를 실행하니 다음과 같은 에러가 발생했습니다.

     [exec] com.~~.SampleTest:...........
     [exec] Error in testSample:
     [exec] java.lang.IllegalArgumentException: dexcache == null (and no default could be found; consider setting the 'dexmaker.dexcache' system property)
     [exec]     at com.google.dexmaker.DexMaker.generateAndLoad(DexMaker.java:359)
     [exec]     at com.google.dexmaker.stock.ProxyBuilder.buildProxyClass(ProxyBuilder.java:252)
     [exec]     at com.google.dexmaker.mockito.DexmakerMockMaker.createMock(DexmakerMockMaker.java:54)

이런 문제가 발생하더구요.

StackOverFlow의 도움을 받아 해결했습니다.

해결 방법은 생성자에 다음과 같은 코드를 추가했습니다.

    public AlbumSetHelperTest() {
        //if target is emulator, it is necessary to set property.
        if ("google_sdk".equals(Build.PRODUCT)) {
            System.setProperty("dexmaker.dexcache", "/sdcard");
        }
    }

이렇게 이슈는 해결했는데, 그렇다면 테스트 코드마다 다 넣어야 할까요?
